analyzer.yml

contexto: estou criando scripts no github actions para postar vídeos automáticamente e trocar horários e ganchos com base em uma análise sem IA dos dados do yooutube e instagram

(parte 1 do script [analyzer.yml]) objetivo 1: 
- analisar métricas
- gerar um arquivo com os resultados (devem conter 2 recomendações de ganchos para a postagem e 2 horários diferentes para as 2 redes sociais diferentes)
- atualisar o "cron:" dos arquivos: "post_I.yml / post_Y.yml"

(parte 2 do script [post_I.yml | post_Y.yml]) objetivo 2:
- ser ativado no horário definido pelo analyze_metrics.py
- usar o arquivo gerado pelo analyze_metrics.py para decidir qual gancho usar e testar
- postar o vídeo de "videos/pending"
- mover o vídeo para "videos/posted" (pensar em um método melhor pois agora há dois script yml para postar os vídeos)


exemplo parte 1 do script:
analyzer.yml > collect_metrics.py > analyze_metrics.py > update_cron.py
